# 高考志愿填报助手 (Gaokao Volunteer Assistant)

一个面向 AI Agent 的 Skill 定义项目，专注于**高考志愿填报**场景。帮助 AI 根据考生的分数、位次、省份、科类、兴趣偏好等信息，提供结构化的院校与专业填报建议，并解释"冲、稳、保"的梯度逻辑。

> ⚠️ **本项目的定位**：这是一个 Skill 定义项目，**不是可运行的 Web 应用**，不包含任何具体的录取分数数据库，也不是志愿填报软件。核心交付物是 `SKILL.md`（AI Agent 工作流定义）和 `references/`（知识参考资料）。

### 核心文件说明

#### SKILL.md

核心文件，包含：

- **YAML frontmatter**：name 和 description，用于 AI Agent 自动匹配触发
- **六步工作流**：
  1. 信息采集（省份、科类、总分、位次、选科、意向等 + 新高考模式识别）
  2. 输入校验（分数位次一致性、选科匹配性、关键字段缺失检查）
  3. 梯度策略分析（冲、稳、保）
  4. 输出推荐表格（6 列标准格式 + 数据来源核对强制项）
  5. 风险提示（免责声明 + 注意事项）
  6. 边界声明（数据来源、能力边界）
- **使用示例与注意事项**

#### references/

参考知识文档，按主题拆分，方便维护和更新：

- **平行志愿规则**：分数优先、遵循志愿、一轮投档解读
- **冲稳保策略**：梯度划分标准、比例建议、常见误区
- **专业级差与调剂**：级差计算方式、服从调剂利弊
- **提前批与特殊类型**：军校/公安/公费师范/强基计划/综合评价/定向招生
- **新高考选科与填报模式**：院校专业组 vs 专业+院校详细对比
- **常见误区**：十大常见错误认知及正确做法

#### examples/

典型测试场景，用于验证 AI 在不同情况下的应对能力：

- 缺少位次时的处理
- 不服从调剂的风险提示
- 提前批/公费师范生咨询
- 选科与意向专业冲突检测

## 能做什么 / 不能做什么

### ✅ 能做的

- 根据用户提供的分数、位次等信息，按"冲稳保"梯度推荐院校和专业方向
- 解释平行志愿规则、专业级差、服从调剂等填报概念
- 根据用户偏好（院校优先/专业优先/分数优先）匹配合适的填报策略
- 提示专业级差、体检限制、单科要求、大小年现象等风险
- 识别新高考不同填报模式（院校专业组 vs 专业+院校）并差异化处理
- 对输入信息进行校验（分数位次一致性、选科匹配性）
- 给出各省志愿数量差异的针对性提醒

### ❌ 不能做的

- **不提供具体院校的实时录取分数线** — 本 Skill 不包含任何院校的实时录取分数数据，也不会编造未经核实的数据
- **不做录取概率的精确计算** — 录取概率只能做大致范围估算，无法精确计算
- **不替代官方数据和招生章程** — 所有推荐均需用户最终以省考试院官方数据和目标院校招生章程为准
- **不回答院校生活类问题** — 宿舍条件、校园环境、食堂情况等问题不在本 Skill 范围内
- **不做出"一定能上"的承诺** — 高考录取存在不确定性，AI 不做任何录取保证
- **不提供当年最新的实时招生计划** — 每年的招生计划以省教育考试院公布的为准

## 安装方法

### 方式一：复制到 Trae IDE 的 skills 目录

```bash
# 将本项目克隆或下载到本地
git clone https://github.com/GZgreywolfy/gaokao-volunteer-assistant.git

# 复制到 Trae IDE 的 skills 目录
cp -r gaokao-volunteer-assistant /path/to/.trae/skills/
```

**Trae IDE 具体操作步骤**：

1. 打开 Trae IDE，进入项目设置
2. 找到 AI Agent / Skills 配置区域
3. 将本仓库克隆到 `.trae/skills/` 目录下
4. 重启 Trae IDE 或刷新 AI Agent 配置
5. 当用户输入高考志愿相关问题时，Skill 将自动触发

### 方式二：Git 子模块（推荐多人协作）

```bash
cd /path/to/你的项目/.trae/skills/
git submodule add https://github.com/GZgreywolfy/gaokao-volunteer-assistant.git
```

### 方式三：手动配置

1. 将本仓库下载到本地
2. 将 `SKILL.md` 文件内容复制到 Trae IDE 的对应 Skill 配置中
3. 将 `references/` 目录下的知识文档导入到 AI Agent 的知识库中

## 使用示例

**用户输入**：

> 我是广东物理类考生，598 分，位次 28000，选科物化生，想去广深，学计算机或电子信息，院校优先。

**Skill 响应摘要**：

1. 确认信息 → 识别广东为院校专业组模式 → 确认冲稳比比例（院校优先 → 积极冲刺 4:3:3）
2. 输入校验（分数与位次一致、选科与专业匹配）
3. 给出推荐表格：
   - **冲**：华南理工大学（部分专业）、暨南大学
   - **稳**：深圳大学、华南师范大学
   - **保**：广州大学、广东工业大学
4. 数据来源核对提醒
5. 附带详细的风险提示和免责声明

## 数据声明

本 Skill **不包含** 任何具体院校的实时录取分数线数据。所有推荐建议均基于：

- 用户提供的分数和位次信息
- 通用填报策略规则
- 用户指定的偏好条件

具体分数线数据请以各省教育考试院和高校官方发布的招生计划为准。

## 注意事项

- 本 Skill 提供的建议仅供参考，不构成填报决策的唯一依据
- 高考政策每年可能调整，请以当年官方文件为准
- 建议结合《志愿填报指南》、高校招生章程等多方信息综合决策
- 推荐结果受限于 AI 的知识边界，可能不包含最新招生变动
