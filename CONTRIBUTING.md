# 贡献指南

感谢您对高考志愿填报助手（gaokao-volunteer-assistant）的关注！本项目的定位是面向 AI Agent 的 Skill 定义项目，核心价值在于**知识准确性、策略合理性和流程规范性**。欢迎各类改进建议和贡献。

---

## 贡献方式

### 1. 知识文档改进（references/）

`references/` 目录下的知识文档是 AI 回答用户问题的参考依据。如果您发现以下问题，欢迎提交改进：

- **政策更新**：高考政策每年可能调整，如发现过时信息请及时更新
- **内容补充**：如某省份的志愿规则有特殊之处，可补充说明
- **错误修正**：如有不准确或误导性的内容，请指出并更正
- **新增主题**：如有重要的高考志愿填报主题未被覆盖，可新增文档

**要求**：
- 内容须有可靠来源（省教育考试院官方文件、教育部政策文件等）
- 保持客观中立，不包含主观评价或商业推广
- 每个文件顶部保留统一格式的免责声明引用
- Markdown 格式规范，标题层级清晰

### 2. 策略逻辑改进（SKILL.md）

`SKILL.md` 是 Skill 的核心定义文件。如果您对工作流程有改进建议：

- **流程优化**：信息采集顺序、校验逻辑、输出格式等方面的改进
- **策略完善**：冲稳比比例、梯度划分标准等策略规则的优化
- **边界明确**：AI 能力边界声明、风险提示内容的补充

**要求**：
- 说明改进理由和预期效果
- 提供具体的修改建议或代码（Markdown 格式）
- 考虑兼容性，避免破坏已有流程

### 3. 测试场景补充（examples/）

`examples/` 目录下的测试场景用于验证 AI 在不同情况下的应对能力：

- 覆盖典型的高考志愿填报咨询场景
- 包含边缘情况和异常输入的处理
- 每个场景包含输入信息和预期的 AI 响应要点

### 4. Bug 报告与功能建议

如果您在使用过程中发现问题或有新功能建议：

- 在 GitHub 仓库提交 Issue
- 请清晰描述问题或建议
- 如有相关截图或示例请一并附上

## 提交规范

### Pull Request 流程

1. Fork 本仓库
2. 创建您的特性分支：`git checkout -b feat/your-feature-name`
3. 提交您的改动
4. 推送到您的 Fork：`git push origin feat/your-feature-name`
5. 创建 Pull Request

### Commit Message 规范

请使用清晰的中文 commit message，分类前缀如下：

| 前缀 | 用途 | 示例 |
|------|------|------|
| `feat` | 新增功能或策略改进 | `feat: 增加输入校验逻辑` |
| `docs` | 文档变更（知识库、README 等） | `docs: 拆分知识库文档` |
| `fix` | 修复错误或不准确的内容 | `fix: 修正平行志愿规则描述` |
| `chore` | 工程化相关（CHANGELOG、CONTRIBUTING 等） | `chore: 新增 CHANGELOG` |
| `refactor` | 结构重组（不改变功能） | `refactor: 重组 references 目录` |

### 文件格式要求

- Markdown 文件使用 `.md` 扩展名
- 编码使用 UTF-8（无 BOM）
- 换行符使用 LF（Unix style）
- 中文和英文之间建议加空格以提升可读性
- 表格对齐规范，建议使用 Markdown 表格格式化工具

## 知识文档编写规范

### 结构模板

```markdown
# 文档标题

> **免责声明**：本文档为通用知识整理，具体政策以各省教育考试院和各高校当年发布的官方招生文件为准。高考政策每年可能调整，请务必查阅最新官方信息。

---

## 正文内容

### 小节标题

内容...

---

## 参考依据

本文档内容基于公开信息整理，旨在提供通用知识参考。具体数据请以各省教育考试院官方发布为准。
```

### 注意事项

- **不编造数据**：不包含任何具体院校的实时录取分数线数据
- **注明来源**：引用政策文件或数据时注明来源
- **客观中立**：不偏袒特定院校或专业
- **时效性**：标注文档适用的年份或说明需要更新

---

## 行为准则

- 尊重不同观点，建设性地讨论问题
- 确保贡献的内容不侵犯第三方权益
- 不将商业推广内容混入知识文档
- 不包含任何形式的歧视性内容
